## Mapping of Commands

| Old (make) | New (just) | Rationale |
|-----------|-----------|-----------|
| `make install-dev` | `just install-dev` | Install dev dependencies |
| `make fmt` | `just fmt` | Format code with ruff |
| `make lint` | `just lint` | Lint code with ruff |
| `make test` | `just test` | Run unit + integration tests |
| `make test-unit` | `just test-unit` | Run unit tests only |
| `make test-integration` | `just test` | Run integration tests (part of default test) |
| `make test-evals` | `just test-evals` | Run eval tests (deterministic only) |
| `make test-evals-full` | `just test-evals --real` | Run eval tests with real LLMs |
| `make check` | `just check` | Run fmt + lint + test |
| `make webapp` | `just webapp` | Launch Streamlit web interface |
| `make test-install` | `just test-install` | Smoke test package imports |

## Notes

1. **Makefile.template** — Still exists in `config/Makefile.template` but is no longer generated by `cli init`. The scaffolder now generates `justfile` only. A `_copy_makefile()` function in `commands_init.py` is already marked as a no-op.

2. **genai-graph vendored Makefile** — Files like `tk_makefile.mk` and `docker.mk` in genai-graph are infrastructure/CI files, not user-facing, so they were left unchanged. These can be migrated separately if needed.

3. **Language references** — Phrases like "make sure", "make this easier", "Make the system", etc. were correctly identified and left unchanged.

4. **No breaking changes** — The `just` task equivalents are already defined in the justfiles and provide the same functionality.
