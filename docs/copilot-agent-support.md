# Copilot Agent Support

genai-tk ships four reusable **Copilot skills** that teach GitHub Copilot how
to extend projects built on the toolkit.  They are generated into new projects
by `cli init` alongside `AGENTS.md` and `.github/copilot-instructions.md`.

---

## Overview

| File | Scope | Purpose |
|------|-------|---------|
| `AGENTS.md` | Project | Architecture overview, import patterns, config reference, skill index |
| `.github/copilot-instructions.md` | IDE | Always-active hints — always loaded by Copilot in VS Code |
| `skills/copilot/*/SKILL.md` | On-demand | Step-by-step procedures Copilot loads when asked to add something |

The skills live in genai-tk's own `skills/copilot/` directory and are
**reusable across all projects** — they ship with the package and are referenced
from the project's `AGENTS.md`.  A project's own domain skills go in
`skills/custom/`.

---

## Skills reference

### `add-cli-command`

**When Copilot loads this skill:** when you ask it to add a CLI command, create
a command group, or extend the CLI.

**What it teaches:**
1. Create a `CliTopCommand` subclass in `<package>/commands/`
2. Register the class in `config/app_conf.yaml` → `cli.commands`

```
skills/copilot/add-cli-command/SKILL.md
```

---

### `add-webapp-page`

**When Copilot loads this skill:** when you ask it to add a Streamlit page,
create a web UI, or add a demo page.

**What it teaches:**
1. Create a Streamlit page in `<package>/webapp/pages/`
2. Use `render_llm_selector()` and `CapturingCallbackHandler`
3. Register the page in `config/webapp.yaml` → `ui.navigation`

```
skills/copilot/add-webapp-page/SKILL.md
```

---

### `add-chain`

**When Copilot loads this skill:** when you ask it to add an LCEL chain,
create a pipeline, or add a runnable.

**What it teaches:**
1. Create a chain module in `<package>/chains/`
2. Call `register_runnable()` at import time so `cli core run` discovers it
3. Common patterns: simple prompt | llm | parser, RAG chain, structured output

```
skills/copilot/add-chain/SKILL.md
```

---

### `add-agent-profile`

**When Copilot loads this skill:** when you ask it to configure a new agent,
add tools, or set up MCP server integration.

**What it teaches:**
1. Add a profile to `config/agents/langchain.yaml`
2. Choose the right agent type (`react` / `deep` / `custom`)
3. Create a custom tool factory in `<package>/tools/`

```
skills/copilot/add-agent-profile/SKILL.md
```

## How Copilot loads skills

Skills are markdown files with a YAML frontmatter block:

```markdown
---
name: add-cli-command
description: Step-by-step procedure to create a new CLI command group ...
---
```

In VS Code with GitHub Copilot, the agent reads the skill index in `AGENTS.md`
and loads individual `SKILL.md` files **on demand** via `read_file` when the
task matches.  This pattern keeps context lean — Copilot only loads the
specific procedure it needs.

Skills can also be referenced explicitly in chat:

```
@workspace /path/to/SKILL.md
```

## Generated project files

### `AGENTS.md` (project root)

Generated by `cli init` from a Jinja2 template.  Covers:

- Project structure and package layout
- Core import patterns (`get_llm`, `def_prompt`, `register_runnable`, etc.)
- Configuration file reference (which YAML file controls what)
- Where to put things: CLI commands, chains, webapp pages, agent profiles, tools
- Coding conventions (Pydantic v2, absolute imports, ruff formatting)
- Skills index — links to each `SKILL.md` file with its use-case description

The generated `AGENTS.md` references the four skills above and can be extended
with project-specific instructions by editing the `## Project-specific notes`
section at the bottom.

### `.github/copilot-instructions.md`

A short (< 20 line) file that is **always loaded** by Copilot in VS Code
(`github.copilot.chat.codeGeneration.instructions`).  It instructs Copilot to:

- Use `uv` to run Python
- Apply Pydantic v2 for all data models  
- Use absolute imports
- Point to `AGENTS.md` for detailed patterns

## Adding project-specific skills

Place additional skill files in `skills/custom/`:

```
skills/
└── custom/
    └── my-domain/
        └── SKILL.md
```

Register them in `AGENTS.md` under the `## Skills` section:

```markdown
| `skills/custom/my-domain/SKILL.md` | How to process my-domain data |
```

Copilot will find them when the relevant task is described.  For deep agents
and Deer-flow, these same skill files can also be _runtime skills_ loaded
by the agent itself — set `skill_directories` in the agent profile:

```yaml
# config/agents/langchain.yaml
langchain_agents:
  profiles:
    - name: MyAgent
      type: deep
      skills:
        directories:
          - ${paths.project}/skills/custom
```

## Using Copilot to extend the project

### Example: add a chain

```
@copilot Add a chain that summarises a long document into three bullet points.
Use the add-chain skill.
```

Copilot will read `skills/copilot/add-chain/SKILL.md`, then:
1. Create `<package>/chains/summarise_chain.py` with an LCEL pipeline
2. Register it with `register_runnable()` so `cli core run summarise_chain "..."` works
3. Suggest a test case

### Example: add an agent page

```
@copilot Create a Streamlit page for a code review agent.
It should let the user paste code and get a review.
```

Copilot will read `skills/copilot/add-webapp-page/SKILL.md` and
`skills/copilot/add-agent-profile/SKILL.md`, then create both
the Streamlit page and the agent profile YAML.

---

## Relationship to `skills/public/` and Deer-flow

The `skills/copilot/` directory is distinct from `skills/public/` (Deer-flow
research and data-analysis skills) and `skills/custom/` (your project-specific
runtime skills).

| Directory | Used by | Purpose |
|-----------|---------|---------|
| `skills/copilot/` | GitHub Copilot Agent | Teach Copilot to extend genai-tk projects |
| `skills/public/` | Deer-flow / deep agents at runtime | Domain knowledge for AI agents |
| `skills/custom/` | Both Copilot and AI agents | Project-specific knowledge |
