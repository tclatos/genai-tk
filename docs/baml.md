# Structured Output with BAML

[BAML](https://docs.boundaryml.com/) (Boundary AI Markup Language) is a domain-specific language
for defining LLM function signatures with strong types.  GenAI Toolkit integrates BAML for
**structured extraction** — running an LLM prompt that is guaranteed to return a typed Pydantic
object rather than free-form text.

---

## Concepts

| Concept | Description |
|---------|-------------|
| **BAML function** | A typed prompt definition (`function ExtractResume(cv: string) -> Resume`) |
| **BAML client** | A Python package generated by `baml-cli generate` containing `types` and `async_client` modules |
| **Config name** | A named configuration entry under `structured.<name>` that tells genai-tk which BAML client package to use |
| **`baml run`** | CLI command to invoke a single BAML function on text input |
| **`baml extract`** | CLI command to batch-process a directory of Markdown files |
| **`BamlStructuredProcessor`** | Python API for programmatic batch extraction with KV-store caching |

---

## Setup

### 1. Write `.baml` function definitions

```
// baml_src/extract.baml

class Resume {
  name string
  skills string[]
  experience Experience[]
}

class Experience {
  company  string
  role     string
  years    int
}

function ExtractResume(cv: string) -> Resume {
  client GPT4o
  prompt #"
    Extract structured resume data from the following text:
    {{ cv }}
  "#
}
```

### 2. Generate the Python client

```bash
# From your project root (where baml_src/ lives)
baml-cli generate

# This creates: baml_client/types.py  baml_client/async_client.py  ...
```

### 3. Configure the client package

```yaml
# config/init/overrides.yaml
structured:
  default:
    baml_client: baml_client   # Python package name (importable)
```

If your generated client lives in a sub-package, use the full dotted path:

```yaml
structured:
  default:
    baml_client: myapp.baml_client
```

### 4. Wire the LLM

BAML uses its own `ClientRegistry` for LLM routing.  GenAI Toolkit maps your standard
model identifier to a BAML client via `create_baml_client_registry()`:

```python
from genai_tk.extra.structured.baml_util import create_baml_client_registry

registry = create_baml_client_registry("gpt_4o@openai")
```

When using CLI commands or `BamlStructuredProcessor`, pass `--llm` / `llm=` and the toolkit
handles the mapping automatically.

## CLI Usage

### `baml run` — Single invocation

```bash
# Run on inline text
uv run cli baml run ExtractResume -i "John Smith; 10 years Python, ML"

# Read from stdin
cat resume.txt | uv run cli baml run ExtractResume

# Select a non-default config
uv run cli baml run ExtractResume -i "..." --config my_config

# Override the LLM
uv run cli baml run ExtractResume -i "..." --llm gpt_4o@openai

# Save to a JSON file
uv run cli baml run ExtractResume -i "..." \
    --out-dir ./output --out-file john_smith.json

# Force overwrite of an existing file
uv run cli baml run ExtractResume -i "..." \
    --out-dir ./output --out-file john.json --force
```

The result JSON is printed to stdout (pretty-printed) unless `--out-dir` and `--out-file`
are both provided.

Output is written to `<out-dir>/<ModelName>/<out-file>` so results from different BAML
functions are automatically organised by their return type.

### `baml extract` — Batch directory processing

Processes every matching Markdown file in a directory and writes a JSON file per input.

```bash
# Basic batch extraction
uv run cli baml extract ./docs ./output --function ExtractRainbow

# Recurse into sub-directories
uv run cli baml extract ./docs ./output --recursive --function ExtractRainbow

# File filters
uv run cli baml extract ./reports ./output \
    --include 'report_*.md' --include 'summary_*.md' \
    --exclude '*_draft.md' \
    --recursive --function ExtractRainbow

# Force re-extraction of all files (ignores manifest)
uv run cli baml extract ./docs ./output --recursive --function ExtractRainbow --force

# Use config variables in paths
uv run cli baml extract '${paths.data_root}/docs' '${paths.data_root}/structured' \
    --recursive --function ExtractRainbow \
    --batch-size 10 --force
```

**Key options:**

| Option | Default | Description |
|--------|---------|-------------|
| `--function TEXT` | required | BAML function name to call |
| `--recursive` | false | Recurse into sub-directories |
| `--force` | false | Re-extract even if file is unchanged |
| `--batch-size N` | 10 | Concurrent tasks per Prefect batch |
| `--include PATTERN` | all | Glob patterns to include (repeatable) |
| `--exclude PATTERN` | none | Glob patterns to exclude (repeatable) |
| `--config TEXT` | default | Config name under `structured.<name>` |
| `--llm TEXT` | default | LLM identifier override |

#### Incremental processing with manifests

`baml extract` maintains a `manifest.json` in the output directory that records:

- Source file hash
- Output JSON path
- BAML schema fingerprint (SHA-256 of the function definition)
- Processing timestamp

On subsequent runs, files are **skipped** if neither the source content nor the BAML
function definition has changed.  This makes large batch re-runs very fast.

The manifest is stored per function name, so different BAML functions writing to the same
output directory don't interfere.

## Programmatic API

### `baml_invoke` — Low-level async call

```python
import asyncio
from genai_tk.extra.structured.baml_util import baml_invoke

result = asyncio.run(
    baml_invoke(
        function_name="ExtractResume",
        inputs={"__input__": "John Smith; SW engineer, 10 yrs"},
        config_name="default",
        llm="gpt_4o@openai",
    )
)
# result is a validated Pydantic instance matching the BAML return type
print(result.name, result.skills)
```

### `BamlStructuredProcessor` — Batch processing with caching

`BamlStructuredProcessor[T]` is a generic Pydantic model for processing many documents
with KV-store caching.

```python
import asyncio
from genai_tk.extra.structured.baml_processor import BamlStructuredProcessor
from baml_client import b  # your generated async client
from baml_client.types import Resume

processor = BamlStructuredProcessor[Resume](
    model_cls=Resume,
    baml_function=b.ExtractResume,   # pass the async BAML method directly
    kvstore_id="file",               # use file-based KV cache
)

document_ids = ["doc_1", "doc_2", "doc_3"]
contents = [open(f).read() for f in ["a.md", "b.md", "c.md"]]

results: list[Resume] = asyncio.run(
    processor.abatch_analyze_documents(document_ids, contents)
)
```

Alternatively, use the **function name** instead of a direct reference (useful when the
BAML client isn't imported at the call site):

```python
processor = BamlStructuredProcessor[Resume](
    function_name="ExtractResume",
    llm="gpt_4o@openai",
    config_name="default",
    force=False,   # set True to bypass cache
)
```

**Caching behaviour:**
- Already-processed documents are loaded from the KV store (keyed by `doc_id`).
- Only uncached documents are sent to BAML — all concurrently via `asyncio.gather`.
- Results are written back to the KV store after successful processing.
- Set `force=True` to bypass the cache and reprocess everything.

## Configuration Reference

```yaml
structured:
  default:                          # config name used with --config / config_name=
    baml_client: baml_client        # importable Python package (generated by baml-cli)

  custom:                           # alternative config for a different BAML client
    baml_client: myapp.other_client
```

`structured.default.baml_client` is required.  Everything else (LLM wiring, KV store)
uses the toolkit's standard configuration.

## How BAML Fits into the Stack

```
BAML .baml files
    │
    ▼  baml-cli generate
baml_client/
    ├── types.py          ← Pydantic models for all BAML types
    └── async_client.py   ← b.ExtractResume(), b.FakeResume(), …
    │
    ▼  loaded dynamically by load_baml_client()
genai_tk.extra.structured
    ├── baml_util.py          ← load_baml_client, baml_invoke, ClientRegistry wiring
    ├── baml_processor.py     ← BamlStructuredProcessor  (KV-cached batch)
    └── baml_prefect_flow.py  ← Prefect flow for directory-level batch extraction
    │
    ▼  CLI layer
genai_tk.extra.structured.commands_baml
    ├── baml run    ← single-item invocation
    └── baml extract ← Prefect-backed batch extraction
```

## Tips and Troubleshooting

**`ValueError: BAML client package not found in config`**  
Make sure `structured.default.baml_client` is set in your `overrides.yaml` and the
package is importable (`uv run python -c "import baml_client"`).

**Schema change not picked up by `baml extract`**  
The manifest uses a SHA-256 fingerprint of the BAML function definition.  If you change
the `.baml` file and regenerate the client, run with `--force` once to invalidate the
manifest, or delete `manifest.json` in the output directory.

**BAML function name not found**  
Function names are case-sensitive and must match the `function` keyword in your `.baml`
file exactly, e.g. `ExtractResume` not `extractResume`.

**Multiple BAML clients in one project**  
Define additional entries under `structured.<name>` and pass `--config <name>` to CLI
commands or `config_name=<name>` to `BamlStructuredProcessor`.
